## 2024-06-26 - Missing Modal Accessibility Attributes
**Learning:** Custom modals (`screenshotModal`, `pickerModal`, `systemMapModal`) in the current UI consistently lack the standard `role="dialog"` and `aria-modal="true"` attributes. Furthermore, icon-only close buttons (using an "X" character instead of an SVG) do not have `aria-label` attributes, rendering them opaque to screen readers. Finally, the modals are missing crucial keyboard accessibility features, specifically dismissing on the `Escape` key, which is standard modal UX behavior.
**Action:** When working on modals or custom overlays, always verify they contain `role="dialog"`, `aria-modal="true"`, dismiss on `Escape`, and that icon-only interactive elements (like close buttons) have an explicit `aria-label`.
