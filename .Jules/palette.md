## 2024-05-19 - Interactive `div` components lacking keyboard accessibility and role semantics
**Learning:** Custom UI modal implementations using generic `div` tags for open/close and trigger behaviors consistently miss native accessibility features, breaking screen reader functionality and keyboard navigation. Even small badges meant to act as triggers must have semantic ARIA roles (`role="button"`) and keydown listeners to mimic true `<button>` behavior.
**Action:** Always add `role="dialog"`, `aria-modal="true"`, and appropriate `aria-label`s to custom modals, and apply `role="button"`, `tabindex="0"`, and `onkeydown` handles (with `event.preventDefault()` for 'Space') to interactive `div` triggers to ensure full keyboard activation and screen reader support without needing to modify existing CSS layouts by converting them to `<button>` elements.
