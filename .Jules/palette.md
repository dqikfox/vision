## 2024-05-24 - Modal Accessibility and Escape Key Support
**Learning:** When custom modals or overlays are implemented without native `<dialog>` tags, they lack crucial accessibility attributes. Modals must have `role="dialog"`, `aria-modal="true"`, and appropriate `aria-label` attributes. Additionally, close triggers should be semantic `<button>` elements (not `div`s) with proper padding/border resets to fit existing styling, and modals must support dismissal via the global `Escape` key for keyboard accessibility.
**Action:** Applied semantic attributes (`role`, `aria-modal`, `aria-label`) to `screenshotModal`, `pickerModal`, and `systemMapModal`. Converted their respective close components to semantic `<button>` tags with correct `aria-label`s and CSS resets. Implemented a global `Escape` key listener to invoke native tear-down functions for each modal type.
