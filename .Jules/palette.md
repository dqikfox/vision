## 2024-05-18 - Accessible Modals Overlay Pattern
**Learning:** When improving accessibility for existing non-semantic custom overlays under strict constraints, adding `role="dialog"`, `aria-modal="true"`, and `aria-label`s makes them screen reader friendly. For non-semantic close buttons (`div` or `span`), adding `role="button"`, `tabindex="0"`, and an `onkeydown` handler supporting both Space and Enter (with `event.preventDefault()` for Space to prevent scroll) allows full keyboard activation without breaking layout by converting to `<button>`. Finally, a top-level `Escape` key event listener that checks for active classes and invokes native close functions enables seamless dismissal.
**Action:** Always apply ARIA modal attributes, ensure keyboard accessibility for non-semantic close buttons, and implement a global `Escape` key dismissal when adding or improving custom overlays.
