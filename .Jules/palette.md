## 2026-04-01 - Modal Keyboard Accessibility and Global Escape Handling
**Learning:** In highly customized, non-framework UIs like `live_chat_ui.html`, modals often lack native HTML `<dialog>` behaviors. When implementing custom modals with `div` overlays, they must explicitly define `role="dialog"` and `aria-modal="true"`. Furthermore, non-semantic close buttons (like a styled `div`) require `role="button"`, `tabindex="0"`, and `onkeydown` event handlers for keyboard (Enter/Space) activation. A global `window` event listener for the `Escape` key is critical to emulate native modal dismissal, but care must be taken to invoke the specific tear-down functions (e.g., `closePicker()`, `closeSystemMap()`) rather than arbitrarily removing CSS classes to ensure proper state management.
**Action:** When building or enhancing custom modals, always enforce the `role="dialog"` pattern, ensure all close actions are keyboard accessible, and implement centralized `Escape` key dismissal that cleanly routes to the modal's specific close function.
