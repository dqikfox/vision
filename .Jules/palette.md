## 2025-05-18 - Modal Accessibility Pattern
**Learning:** Custom modals in this application lack basic semantic structure and keyboard navigation. Using raw `<div>` tags for close buttons breaks focus states and screen reader utility. Inline class manipulation (`classList.remove('open')`) causes inconsistent state management across different interaction methods (click vs. keyboard).
**Action:** When creating custom modals or overlays, always include `role="dialog"` and `aria-modal="true"`. Convert interactive `<div>` elements to semantic `<button>` tags with explicit `aria-label`s and style resets (e.g., `padding:0; border:none; background:none;`). Abstract dismissal logic into dedicated close functions (e.g., `closeModal()`) and attach a global `Escape` key event listener to gracefully trigger them.
