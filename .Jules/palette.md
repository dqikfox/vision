## 2025-02-23 - Custom Modals Missing Global Escape Binding and Semantic Constraints
**Learning:** Custom UI overlay panels (`#pickerModal`, `#systemMapModal`, `#screenshotModal`) in this application heavily rely on manual JS `.classList.add('open')` for visibility without native `<dialog>` tags. When creating or modifying these overlays, they consistently lack `Escape` key dismissal handlers out-of-the-box and require non-semantic `<div role="button">` wrappers for close buttons due to strict global CSS constraints on standard `<button>` tags.
**Action:** When adding or maintaining full-screen overlays in `live_chat_ui.html`, always explicitly attach a top-level `window.addEventListener('keydown')` for the `Escape` key that invokes the specific `closeComponent()` function. Any non-semantic `div` or `span` used as a close trigger must explicitly receive `role="button"`, `tabindex="0"`, `aria-label`, and an `onkeydown` handler trapping both `Enter` and `Space` (with `event.preventDefault()` to stop page scroll).
