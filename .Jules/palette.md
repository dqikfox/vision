## 2024-05-18 - Semantic Buttons and Keyboard Focus in Custom Modals
**Learning:** In custom UI designs (like modal windows or inline dynamic lists), close controls (`.modal-close`) and list item controls (`.rm`) are often mistakenly built using `<div>` or `<span>` elements instead of `<button>`s. This breaks screen reader discoverability and keyboard navigation. Additionally, icon-only buttons frequently lack `aria-label`s, rendering them invisible or confusing to assistive technologies. Custom styling often omits clear `focus-visible` outlines, hiding where keyboard focus is.
**Action:** Always convert interactive `div`/`span` elements to semantic `<button>` tags, attach descriptive `aria-label`s to icon-only buttons, and ensure a global `button:focus-visible` CSS rule exists in the application so keyboard users can navigate confidently.
