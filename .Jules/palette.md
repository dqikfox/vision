## 2025-03-05 - Add ARIA Roles and Escape Key Support to Modals
**Learning:** Native `<div>` elements used as close buttons fail accessibility checks because they lack semantic meaning and keyboard focus. Converting them to `<button>` with explicit `aria-label`s and adding `role="dialog"` to the container significantly improves screen reader compatibility. Additionally, a global `Escape` key listener that taps into existing `.open` state and uses native close functions provides essential keyboard accessibility for modal dialogs.
**Action:** When implementing custom modals, always use semantic `<button>` tags with reset styles (`padding:0; border:none; background:none;`) for close actions. Ensure modals use `role="dialog"` and `aria-modal="true"`. Finally, always implement a global `Escape` key listener that checks the active state and gracefully delegates to the component's teardown function (e.g. `closePicker()`).
