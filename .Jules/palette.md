## 2025-02-28 - Semantic Modal Accessibility Pattern
**Learning:** Found custom modal implementations in `live_chat_ui.html` missing critical accessibility hooks. They lacked semantic ARIA attributes (`role="dialog"`, `aria-modal="true"`) to capture screen readers, used inaccessible `<div>` tags for close buttons, missed explicit `aria-label`s on icon-only close buttons, and lacked keyboard interaction (no `Escape` key support to dismiss).
**Action:** Always wrap custom modal container elements with `role="dialog"` and `aria-modal="true"`. Ensure close actions are bound to semantic `<button>` elements with clear `aria-label`s. Implement global `Escape` key event listeners to seamlessly dismiss active overlays to ensure keyboard accessibility matches expected interaction patterns.
