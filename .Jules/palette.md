## 2024-03-21 - [Modal Accessibility and Keyboard Navigation Constraints]
**Learning:** Adding keyboard accessibility to non-semantic elements (like `div` or `span`) used as close buttons for modals requires careful implementation (`role="button"`, `tabindex="0"`, `aria-label`, and `onkeydown` handlers for Enter and Space) rather than changing the tag to `<button>`, which may break existing CSS layouts. Furthermore, modals themselves should implement semantic ARIA roles (`role="dialog"`, `aria-modal="true"`, `aria-label`) to be properly recognized by screen readers. A global `Escape` key listener is a standard UX expectation for dismissing modals, and it's best practice to invoke the native close functions of the modals rather than manipulating CSS classes directly.
**Action:** When creating or updating custom modals, always add `role="dialog"`, `aria-modal="true"`, and a descriptive `aria-label`. For close buttons, if using non-semantic elements, apply the necessary ARIA attributes and keyboard event handlers. Implement a global `Escape` key listener that safely invokes component-specific close functions.
