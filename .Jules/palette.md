## 2024-05-24 - Accessibility for icon-only actions
**Learning:** The application extensively uses non-semantic elements (`div`, `span`) with click handlers for interactive UI components like close buttons and delete actions. This breaks keyboard accessibility and screen reader support. Simply converting them to `<button>` elements can break existing layouts and requires extensive inline CSS, which violates the strict constraint against custom CSS.
**Action:** When improving accessibility for existing non-semantic interactive elements under strict CSS constraints, instead of converting them to `<button>`, add the `role="button"` and `tabindex="0"` attributes. Also, ensure you add `onkeydown` event handlers to support keyboard activation (Enter and Space keys), and add `aria-label`s for screen reader support.
