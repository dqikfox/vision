## 2024-05-18 - Custom Modal Accessibility & Keyboard Support
**Learning:** Custom UI modal elements (`#screenshotModal`, `#pickerModal`, `#systemMapModal`) in the current app often lack standard ARIA semantic attributes (`role="dialog"`, `aria-modal="true"`) and semantic screen reader descriptions. Close buttons frequently lack `aria-label`s, and `Escape` key support for dismissal is consistently missing, requiring explicit global event handlers to improve keyboard accessibility and navigation compliance.
**Action:** When working on custom modals, always enforce ARIA roles, provide explicit text via `aria-label`s on icon-only close buttons, ensure focus visibility, and add global `Escape` key event listeners connected to specific tear-down functions (`closeComponent()`) to ensure keyboard users can safely dismiss overlays.
