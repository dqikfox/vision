## 2026-07-20 - Custom Modal Accessibility
**Learning:** Custom UI overlay panels (`#pickerModal`, `#systemMapModal`, `#screenshotModal`) in `live_chat_ui.html` relied on manual JS `.classList.add('open')` for visibility without native `<dialog>` tags. They lacked semantic attributes (`role="dialog"`, `aria-modal="true"`) and proper keyboard accessibility. Converting non-semantic elements like `div.modal-close` to `<button>` can break the existing layout.
**Action:** When implementing modal accessibility under strict CSS constraints, keep the non-semantic tags but add `role="button"`, `tabindex="0"`, `aria-label`s, and `onkeydown` event handlers for keyboard activation (Enter and Space keys). Make sure to include `event.preventDefault()` for the Space key to prevent page scrolling. Ensure a global `Escape` key event listener is attached at the top level or within `DOMContentLoaded` to handle dismissal, and invoke the native JS close routines (e.g., `closePicker()`) instead of manually removing classes.
